# Nonsense
<!-- cspell: disable -->
Nonsense is a package for making synthetic languages and translating text to and from those languages. It's not proper
languages, simple syllable search and replace.

## Setup

Install [`mise-en-place`](https://mise.jdx.dev/) and then run these commands:

```bash
mise trust
mise install
pnpm install
nx run-many -t prune
```

This sets up the monorepo and builds installable CLI artifacts:

* [`syllables-cli`](./apps/syllables-cli/): Counts syllable frequencies in text input and outputs a ranked list.
* [`syllable-map-cli`](./apps/syllable-map-cli/): Builds a syllable substitution map from a ranked syllable list (e.g.
  the output of `syllables-cli`).
* [`synthetic-language-cli`](./apps/synthetic-language-cli/): Translates text to or from a synthetic language using a
  syllable substitution map (e.g. the output of `syllable-map-cli`).

## Usage

Run the packaged CLIs directly using Node:

```shell
# Analyze syllable frequencies
node ./apps/syllables-cli/dist/main.js input.txt

# Build a syllable substitution map from ranked syllables
node ./apps/syllable-map-cli/dist/main.js ranked.csv

# Translate text to or from a synthetic language
node ./apps/synthetic-language-cli/dist/main.js --map map.csv --direction to-synthetic input.txt
```

This example shows how to generate a complete synthetic language and translate text:

```shell
unzip examples/english.zip -d examples/
node ./apps/syllables-cli/dist/main.js -l 50000 examples/english.txt > examples/ranked.csv
node ./apps/syllable-map-cli/dist/main.js examples/ranked.csv > examples/zorkish.language-map
node ./apps/synthetic-language-cli/dist/main.js --map examples/zorkish.language-map --direction to-synthetic examples/the-verdict.txt > examples/zorked.txt
node ./apps/synthetic-language-cli/dist/main.js --map examples/zorkish.language-map --direction from-synthetic examples/zorked.txt > examples/unzorked.txt
```

## CI/CD

GitHub Actions drives the release pipeline:

* Pull requests run `pnpm exec nx run-many -t test`.
* Pushes to `main` run Commitizen and create the next `v*` tag. This workflow currently uses a patch bump on each non-`bump:` push to `main`.
* New `v*` tags create GitHub releases with autogenerated release notes.
* Published GitHub releases publish these packages to `https://npm.pkg.github.com`: `@mhvelplund/syllables-core`, `@mhvelplund/synthetic-language-core`, `@mhvelplund/syllables-cli`, `@mhvelplund/syllable-map-cli`, and `@mhvelplund/synthetic-language-cli`.

### Required repository settings

* Add a `COMMITIZEN_PAT` Actions secret containing a classic personal access token with repository write access. The versioning workflow must use a PAT so the pushed tag can trigger the downstream release workflow.
* Give `main` branch protection rules that funnel changes through pull requests if you want "push to main" to mean "merged to main" in practice.

### GitHub Packages notes

GitHub Packages only accepts scoped npm package names, so published packages use the `@mhvelplund/*` scope. The release workflow publishes the libraries from their source package roots and rewrites the CLI `dist/package.json` files just before publish so the packaged CLIs depend on the registry versions instead of local `workspace_modules`.
